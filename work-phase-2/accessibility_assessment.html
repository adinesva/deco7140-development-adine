Accessibility Assessment

WCAG Guidelines tailored for client

Here are some WCAG guidelines and success criteria tailored for the client.

Text Alternative

Alternative text for non-text content, user using assistive technologies like screen readers can still understand what it is.

✅ This criteria is relevant as there will be various audience, especially from international with different languages, different tech comfort levels, and different abilities. Some may have reading disabilities and prefer text description.

Meaningful Sequence

To cater users using screen readers, braille displays, or text-to-speech so they can read the content in order.

✅ This criteria is relevant as it ensures that information is read in the right order, supports screen readers and translation tools, and helps newcomers follow important instructions without confusion.

Avoid Color-only Cues

Color alone cannot be the only way you convey information, show an action, or highlight something important.

Perceivable Text Contrast

Text and images of text must have enough color contrast against their background so people with low vision can still read it.

✅ This criteria is relevant as it make sures all user can read the content without difficulties, regardless of vision ability, device quality, or environment.

Resize Text Without Loss

Helps user with low vision and resizing for mobile-first design.

✅ This criteria is relevant as it make sures all users especially with low vision, non-native English speakers, and mobile users can zoom or enlarge text without breaking the layout or losing important content.

Keyboard Accessibility

Able to navigate, operate controls and activate features by only using a keyboard.

✅ This criteria is relevant as supporting keyboard use ensures compliance to Disability Discrimination Act (DDA). This ensures all users including those with disabilities, screen reader users, and people relying on alternative input methods can access the site.

Focus Visible

The currently active element (button, link, input, etc.) must have a clear and visible focus indicator.

✅ This criteria is relevant as this ensures navigation is predictable, usable and trustworthy. Because users will rely in our site to search important information. If the menus, forms, or buttons don't show focus, users literally won't know what they're selecting.

Reading Level

Provide simple alternative for complicated content.

Consistent Navigation

Users don't have to relearn where the menu is on each page, navigation is intuitive.

✅ This criteria is relevant as this ensures menus and page structure stay stable, intuitve and predictable. This makes the site easier to navigate, and minimizes users' confusion.

Consistent Identification

Components should be identified consistently by users across the site.

✅ This criteria is relevant as this ensures users, especially international newcomers navigating in a second language or with assistive tech always recognize the same function by the same name or icon, making the site predictable, trustworthy, and easy to use.

Error Identification

Error should be identified and described to the user by text and color.

✅ This criteria is relevant as this ensures when users make mistakes in forms or inputs, they get clear feedback about what went wrong and how to fix it.

Labels and Instructions

Labels should be described so that users know what to expect. Screen readers rely on proper labels to announce form fields.

✅ This criteria is relevant as this ensures every input field tells users what to input and how to input it, this is needed to reduce error, support screen reader users, and makes the experience easier for people with cultural difference and tech literacy.

Error Prevention and Recovery

Protection for sensitive data and financial error, to prevent accidental error.
